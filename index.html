What is shadeDB?

shadeDB stops trying to be everything and focuses on raw speed, autonomy, and developer ergonomics — a field→value storage engine with a compact SNL (Structured Native Language) interface and a TLS/HTTPS API gateway.

Initially developed during the evolution of Shadecreed, the engine was designed to tackle real-world challenges like JSON duplicate-key resolution and strict memory predictability. The prototype proved surprisingly fast—able to compile and return queries within seconds, even though it wasn’t production-ready. That success inspired a new mission: to build a lightweight database capable of compiling disk state and delivering query results in sub-millisecond time.

Engine Architecture & SNL

Small binary-tree storage engine, custom indexing, and a minimal Structured Native Language (SNL) for simple commands.

Storage Engine

  • Binary-tree based storage structure
  • Custom indexes for predictable latency
  • Write-Ahead Log (WAL) optimized for overwrites + append
  • Field → Value native model

Security & API

  • TLS-encrypted HTTPS endpoints
  • Auth tokens & multi-tenant isolation
  • Field-level sensitivity controls and masking
  • Query pre-filtering to reject malformed/malicious inputs
